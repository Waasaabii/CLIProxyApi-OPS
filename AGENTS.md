# AGENTS 指南

## 基本要求
- 所有交流、分析、实现说明与验证结论必须使用简体中文。
- 默认优先长期可维护方案，不为了临时方便绕开工程约束。

## 工作区边界约束
- 所有代码、脚本、测试、构建、日志、临时文件、导出文件必须默认落在当前工作区内，不允许写到工作区外。
- 严禁默认使用系统级临时目录或工作区外路径，包括但不限于 `/tmp`、`/private/tmp`、`/var/tmp`、用户家目录、桌面目录。
- 需要临时目录时，统一放在工作区内的 `./.tmp/`、子项目内的 `./.tmp/`，或与目标文件同目录的临时文件。
- 传入路径参数时，必须先校验是否位于工作区内；发现路径越界时必须立即拒绝执行，而不是自动创建或继续运行。
- 除非用户明确批准，否则不允许通过软链接、相对路径跳转、环境变量覆写等方式绕过工作区限制。

## 命令与测试约束
- 运行命令时优先使用项目目录作为 `workdir`，避免把输出散落到工作区外。
- 重定向输出文件、测试快照、二进制产物、调试日志默认写入项目内目录，例如 `./.tmp/`、`./dist/`、`./logs/`。
- 清理操作只允许删除当前任务产生的工作区内文件；如果要删除工作区外内容，必须先获得明确确认。

## 高风险操作确认
- 以下操作必须先获得明确确认：删除工作区外文件、删除系统临时目录内容、批量删除 Docker 资源、修改系统级环境、访问生产环境。

```text
⚠️ 危险操作检测！

操作类型：[具体操作]
影响范围：[详细说明]
风险评估：[潜在后果]

请确认是否继续？[需要明确的“是”、“确认”或“继续”]
```
