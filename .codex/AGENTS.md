# Codex 工作区约束

## 语言要求
- 所有交流、分析、实现说明、测试结论必须使用简体中文。

## 工作区边界
- 默认禁止把任何文件写到当前工作区之外。
- 严禁默认使用系统临时目录或工作区外路径，包括但不限于 `/tmp`、`/private/tmp`、`/var/tmp`、用户家目录、桌面目录。
- 临时文件、日志、构建产物、测试输出统一放在工作区内，例如 `./.tmp/`、`./dist/`、`./logs/`，或目标文件同目录。
- 所有路径参数在执行前都必须校验是否仍位于工作区内；如果路径越界，必须直接报错并停止执行。
- 不允许通过环境变量覆写、相对路径跳转、软链接等方式绕过工作区限制。

## 命令执行
- 运行命令时优先使用工作区内目录作为 `workdir`。
- 重定向输出文件时，必须显式落在工作区内，不允许省略路径后依赖系统默认目录。

## 清理与危险操作
- 清理操作只允许删除本任务产生的工作区内文件。
- 删除工作区外文件、系统临时目录内容、批量删除 Docker 资源，必须先获得用户明确确认。
